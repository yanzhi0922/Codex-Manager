# cockpit-tools 借鉴边界与产品对照

## 背景

用户希望 Codex-Copilot 参考 [cockpit-tools](https://github.com/jlcodes99/cockpit-tools) 的产品成熟度继续改进。调研时看到 cockpit-tools 当前定位为通用 AI IDE 账号管理工具，覆盖多平台账号、实例、配额、唤醒与设置等视图，最新 release 为 `v0.23.9`。

## 许可边界

cockpit-tools README 中声明项目采用 [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/) 许可证，并限制商业使用、商业集成和再分发。因此 Codex-Copilot 不复制 cockpit-tools 的源码、资源、图标、样式或文案。

本次只借鉴产品结构：

- 平台能力矩阵
- 账号与实例分离
- 配额与状态统一仪表盘
- 多实例生命周期管理
- 设置、发布、诊断入口集中化

## 对照结论

| 维度 | cockpit-tools 的强项 | Codex-Copilot 当前状态 | 本次落地 |
| --- | --- | --- | --- |
| Dashboard | 多平台状态总览、配额进度和快捷操作 | 已有账号池、额度、请求统计 | 新增“能力驾驶舱”作为产品地图 |
| Account | 多平台账号管理、批量操作、配额展示 | Codex 账号池较完整 | 保持 Codex 深度，不急于铺开多平台 |
| Instance | 多开实例、独立目录、启动/停止 | 暂未形成实例模型 | 标记为下一阶段，并明确实例目录、启动命令、绑定账号三件套 |
| Platform Matrix | 多 AI IDE 支持矩阵 | 当前以 Codex 为核心 | 增加只读平台矩阵，先记录边界和状态 |
| Quota | 多平台配额来源统一展示 | Codex 额度、请求日志和平台 Key 用量已具备 | 把配额纳入能力驾驶舱说明 |
| Release / Ops | release preflight、版本同步、构建文档 | 已有 CI 和 release workflow | 能力驾驶舱把运维视角纳入产品质量面板 |

## 产品原则

1. Codex-Copilot 先做 Codex 的深度闭环，再做跨工具广度。
2. 对其他 AI IDE 先做只读探测和实例边界，不直接写入凭证。
3. 所有会破坏用户本地客户端状态的动作，必须先有备份、预览和回滚方案。
4. 多实例能力优先抽象为“实例目录 + 启动参数 + 绑定账号 + 生命周期”，避免和账号池、会话迁移耦死。

## 后续建议

1. 为 Codex 实例建立最小模型：`id`、名称、工作目录、Codex home、绑定账号、启动命令、状态。
2. 扩大只读平台探测：继续补充各平台安装位置、配置版本和可展示状态，但仍不直接写入第三方客户端凭证。
3. 把会话管理的工作区字段和请求日志的项目路径关联，形成项目级时间线。
4. release workflow 加入更明确的 preflight 输出，减少本地/CI 差异。
